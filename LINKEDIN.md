>
> So I built **Nexora AI Finance Department** — a full-stack web app with six specialized AI agents, each an expert in a different domain:
>
> 🏦 **CFO Agent** — executive summaries, board narratives, risk commentary
> 📊 **FP&A Agent** — variance drivers, forecast methodology, accrual checklists
> 📋 **Procurement Agent** — contract pipeline, vendor negotiation tactics
> 👷 **External Labor Agent** — contractor burn rates, SOW compliance
> 👥 **Headcount Agent** — open reqs, fill rate, salary budget
> 💡 **CIO Finance Partner** — cloud spend, FinOps strategy, talking points

> - Month-over-month trend analysis
> - Recommended actions with owners and due dates

> → Realistic mock dataset: 200+ financial records across 5 data domains
> → Full-stack with server components, mobile-responsive layout, API route for Claude upgrade

## Interview Talking Points

### "Tell me about a project you're proud of."

> "I built Nexora AI Finance Department — a full-stack web app that simulates an IT Finance team using six AI agents. What I'm most proud of is that it's genuinely useful. The FP&A Agent's variance driver analysis cites specific dollar figures, names root causes, and suggests prioritized actions — it reads like something a senior analyst actually wrote. That quality comes from the finance expertise I put into it, not from the AI itself.
>
> Technically, I built the whole stack: a Next.js App Router application, TypeScript throughout with strict mode, a custom routing engine that scores keyword matches to select the right response handler, and a character-streaming UI that makes the interaction feel intelligent. The whole thing runs without a paid API key — I designed it mock-first — but the architecture is wired to upgrade to Claude with a single environment variable."

### "How does your finance experience make you a better engineer?"

> "When I was building the FP&A Agent, I had to decide what the variance driver response should contain. A generic engineer would probably output a list of numbers. Because I've written real variance commentary for CFOs, I knew the response needed: the root cause (not just the symptom), a dollar-level BU breakdown, a mention of the favorable offsets that partially explain the overrun, and two or three prioritized actions with owners. That specificity is hard to specify without having done the work.
>
> Finance gave me experience working with complex, interconnected datasets where errors have real consequences. That shapes how I think about data integrity, type safety, and testing — which made me a better TypeScript engineer."

### "Why did you build this instead of just using ChatGPT?"

> "That was actually a core design decision. ChatGPT doesn't know my actuals. If you ask it 'what's driving our variance?' it either refuses or makes something up. My agents know the actual numbers — the $847K variance, the AWS overage, the four over-budget contractors — because those figures are programmatically injected into every response. The AI's job is language; the data layer provides the facts. That separation is what makes the responses trustworthy enough to show to a CFO."
